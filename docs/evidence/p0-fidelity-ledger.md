# P0 实现保真清单

基线设计是 [`docs/product-technical-design.md`](../product-technical-design.md)。本清单以当前代码为准，区分“可演示实现”“部分实现”和“未实现”，防止把规划、数据模型或占位入口当成已交付能力。

## 状态定义

- **可演示实现**：存在用户可操作页面/接口、确定性实现或降级，并有对应自动化测试文件；仍可能缺少生产账号、云部署或手工验收。
- **部分实现**：关键子链路可运行，但没有满足设计文档的完整持久化、角色、数据或跨端闭环。
- **未实现**：只有设计、基础数据表、静态文案或未来接口设想，不能作为现场功能演示。

## 演示底线

1. 不用截图冒充动画，不用预录输出冒充 Python 执行。
2. 不把浏览器 localStorage 称为生产数据库或跨端学习档案。
3. 不把 Core API 已有数据表称为网页已经接入的功能。
4. 不把项目内 8 门原创课程和两个主题的小型参考目录称为官方教材或完整权威知识库/RAG。
5. 不把浏览器挑战通过称为正式成绩或安全的服务端判题。
6. 不把设备在线状态称为共享会话或机器人多模态教学闭环。
7. 不把 Git/Vercel 配置文件称为已经完成云部署。
8. 没有最新运行证据时，不报告“全部测试通过”或响应时间/准确率等数字。

## 发布准入建议

比赛 P0 可在完成自动化、真实浏览器、Office 文件、Gemini、Redis、Core HTTPS、OrangePi WSS 和移动端手工验收后发布为“可演示原型”。面向真实学校试用前，身份与授权、正式教材/RAG、服务端学习档案、内容审核、监护/同意、数据删除、独立设备凭证和生产监控都是阻断项。
