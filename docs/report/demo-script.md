# 比赛演示脚本（8-10 分钟）

本脚本只使用当前代码存在的功能。它是待排练脚本，不是已经执行的演示证据；正式录屏后应在 [`docs/evidence/index.md`](../evidence/index.md) 补充文件名、时间戳和 release commit。

## 演示前准备

### 环境

- 使用一个新的浏览器 profile 或确认演示站点 localStorage 是预期状态。
- 预先打开学习首页、`/lab` 和 `/progress`，但不要用截图替代真实操作。
- 验证 Gemini 文本、图片和录音；允许失败时走种子内容降级。
- 验证 DOCX/PPTX 下载目录和 Word/PowerPoint 能实际打开文件。
- 验证 Redis、Core HTTPS、OrangePi WSS 和页面右上设备状态。
- 麦克风/扬声器音量先低音量测试，关闭通知和其他录音软件。
- 准备一张无个人信息的教学图片，JPEG/PNG/WebP 且小于 4 MiB。
- 固定演示 commit、Vercel deployment 和 Core 镜像；不要现场改环境变量。

### 证据记录

录屏同时记录：日期、操作者、浏览器版本、网页 deployment、Core deployment、commit SHA 和网络。所有 Token/API Key/数据库地址在录屏中隐藏。

## 连续演示流程

### 0:00-0:40 定位与架构

讲解：

> Mambo 是面向 K12 AI 通识教育的学习工作台，网页是主要教学产品，OrangePi 是可选的陪伴终端。大模型负责适龄解释和结构化绘本，动画、评分、代码检查和设备命令由确定性程序负责。

画面：学习工作台三栏，指出课程路径、对话课堂、教学画布和右上机器人状态。不要宣称已有教师端或正式教材库。

### 0:40-1:40 小学低年级语音与适龄对话

1. 选择“小学低年级”。
2. 选择“图片标签小侦探”或对应图像分类课程。
3. 点击录音，问：“电脑是怎么认出小猫的？”停止录音，确认转写文字后发送。
4. 展示短句、故事化解释和一个具体互动问题。
5. 点击回答的朗读按钮，播放一小段后停止。

讲解重点：录音经 Gemini 转写，回答朗读由浏览器 `speechSynthesis` 完成；不是实时双工。发送后的文字转写与助手回答会按课程有界保存在本浏览器，录音二进制不会保存。

### 1:40-2:30 单图提问

1. 选择准备好的无个人信息图片。
2. 输入“请告诉我模型会先看哪些线索”，发送。
3. 展示图片预览和流式回复。
4. 再发一条纯文字问题，说明旧图片不会被重复发送。
5. 刷新页面，展示完整文字轮次恢复，同时说明图片不会从历史恢复。

讲解重点：只允许最后一条学生消息附一张 JPEG/PNG/WebP，浏览器和服务端都限制 MIME/大小；每门课只保留最多 20 条/20,000 字符的完整文字消息，没有云同步。

### 2:30-3:35 互动绘本

1. 打开教学画布“绘本”标签。
2. 翻页并点击朗读。
3. 回答页内问题，展示即时解释。
4. 点击保存，显示本机保存版本数。
5. 点击重新生成；成功时说明是结构化 AI 输出，失败时展示种子降级仍可继续。
6. 点击“查看最近保存”演示回看。

讲解重点：绘本必须是 4-8 页并通过 Schema；插图是项目内资产。当前作品仅保存在这台浏览器，不是云作品集。

### 3:35-4:25 同主题学段切换

1. 切换到高中。
2. 选择“图像分类系统审计”或“排序算法实验：冒泡排序”。
3. 对相同概念提问，展示先结论、术语、参数/复杂度/边界的高年级表达。
4. 对照刚才低年级回答说明差异来自确定性学段策略与课程数据共同进入提示词。
5. 在课程区展示“事实依据”，打开 `[1]` 对应来源；说明锚点课把同一版本化事实/来源加入提示并要求 `[S#]`。

不要声称已经接入年级教材或完整 RAG；当前是四学段原创课程加两个主题的小型静态参考目录。

### 4:25-5:25 两类交互动画

1. 选择冒泡排序课程，打开“动画”。
2. 依次操作单步、播放、暂停、2x、重置；指出比较数、交换数和已就位区间。
3. 切换图像分类/神经网络课程，再次打开“动画”。
4. 单步点亮输入像素、隐藏特征和类别概率，指出“概率是模型预测，不是事实”。

讲解重点：动画是状态机真实运行，不是 GIF、截图或模型生成任意网页代码。

### 5:25-6:05 Word/PPT 教学材料

1. 打开“资源”标签。
2. 下载 Word 讲义和 PowerPoint 课件。
3. 用真实 Office 应用打开一个锚点课文件，快速展示学习目标、讲解、活动、动画步骤、练习、总结和参考来源 URL。
4. 说明没有来源目录的课程会明确标记“项目原创种子课程，尚未绑定正式教材”。

讲解重点：文件由固定课程结构真实生成；学生材料不含答案。视频目前只是录制建议，没有上传/生成平台。

### 6:05-7:15 游戏练习与个性化

1. 打开“练习”标签。
2. 单选题故意答错，展示纠错反馈和提示。
3. 重试并答对。
4. 演示排序题的上移/下移，或代码追踪题。
5. 打开“学习进度”，展示知识点证据、下次复习和推荐原因。
6. 修改一个兴趣，展示推荐仍以当前学段和学习证据为主，兴趣只作同等候选排序。

讲解重点：客观题由程序确定性评分；本机掌握度是形成性演示，不是正式成绩。

### 7:15-8:25 Python 编程实验

1. 打开“编程实验”。
2. 等待“Python 已就绪”。
3. 在冒泡排序模板补全核心代码并运行，展示真实 stdout 和挑战结果。
4. 切换图像分类模板，展示学段化任务/提示。
5. 可选：制造一个语法错误，展示错误信息，再重置模板。
6. 指出通过同一挑战版本不会重复增加证据。

讲解重点：代码在无主站同源身份的隔离 iframe Worker 中由真实 Pyodide 运行；它不是正式服务端考试沙箱，只记低权重形成性证据。

### 8:25-9:10 机器人与网页降级

1. 展开右上设备状态，展示设备名、最后心跳、`ping`/`get_status` 以及代理从 `/dev` 节点检测到的摄像头/音频/屏幕/NPU 能力；说明节点存在不等于刚完成了一次硬件自检。
2. 若现场安排了离线演练，停止代理前先说明，页面应转为“网页模式”，课程仍可使用；随后恢复代理观察自动重连。
3. 强调 Web 只读设备状态，不下发 Shell 或任意控制命令。

不要声称网页和机器人已经共享对话/作品；当前只打通 WSS 设备状态链路。

### 9:10-9:50 收束与边界

讲解：

> 当前原型已经把适龄对话、图片与语音、版本化锚点来源、Office 材料、两类动画、互动绘本、Python 和三类练习放进同一学习闭环。下一步不是增加静态入口，而是把小型静态 grounding 扩展为正式教材 RAG，接入账号/教师审核、服务端学习档案，以及机器人语音、屏幕和结构化 NPU 视觉事件。

最后显示架构图或报告的保真清单，主动说明没有完成的登录、教师 CMS、完整教材 RAG、视频、共享会话和生产部署证据。

## 故障降级表

| 故障 | 现场动作 | 仍可演示 | 不应做 |
|---|---|---|---|
| Gemini 不可用 | 说明外部 Provider 失败；不连续重试 | 固定课程、两类动画、种子绘本、DOCX/PPTX、练习、Python | 不把预录回答冒充实时模型 |
| Redis 不可用 | 展示 503 故障关闭，恢复预先配置的 Redis | 非 AI 内容全部可用 | 不临时关闭生产限流或把 Secret 写前端 |
| 麦克风权限失败 | 改用文本输入，说明浏览器权限边界 | 文本、图片与其他内容 | 不进入系统设置暴露个人通知/设备信息 |
| speechSynthesis 无中文语音 | 停止朗读演示 | 所有视觉内容 | 不宣称项目内置了独立 TTS 模型 |
| Pyodide CDN 不可达 | 展示明确“环境需重试”，切到动画/练习 | 除 Python 外的学习闭环 | 不用预录终端输出冒充运行 |
| Core/WSS 不可用 | 展示“网页模式” | 完整网页教学 | 不现场改 Token、sudo 或重装系统 |
| Office 打不开 | 保留下载文件和 API/OOXML 自动化证据，标记手工验收失败 | 网页其他能力 | 不只展示文件名就声称 Office 已验收 |
| Vercel 现场网络差 | 使用预先验证的本地构建和 Core 局域网方案 | 本地完整网页 + 板端 | 不临时部署未测试 commit |

## 禁止表述

- “已经接入官方教材/知识库”
- “所有学习记录都在数据库并跨端同步”（当前只有本浏览器的有界文字对话和学习状态）
- “机器人可以继续网页里的同一段对话”
- “Pyodide 是绝对安全的正式判题沙箱”
- “视频、教师 CMS、登录和班级分析已经完成”
- “Vercel、Redis、PostgreSQL 和公网 WSS 已部署”，除非有本次 release 的真实证据
- “全部测试通过”，除非刚运行了完整发布命令并保存日志

## 正式录屏后补录

在证据矩阵为每个 R01-R16 填入：

```text
visual_ref: demo-v1.mp4
demo_timestamp: 00:00-00:00
owner/status: <操作者> / verified on <commit SHA>
```

截图、录屏、生成文件和日志应与同一个 commit 绑定，失败步骤也应如实记录。
